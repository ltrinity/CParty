113double get_feature_counts_restricted (char *sequence, char *structure, double *c, double &f, int reset_c, int ignore_dangles, int ignore_first_AU_penalty);
114// This function computes the c vector and the f value of a linear energy function c' x + f
115// structure contains the following characters:
116// - left and right parentheses mean base pairs, to be considered in the energy model
117// - dots mean unpaired bases, to be considered in the energy model
118// - a sequence of x's means ignore that region from the energy model
119// - left and right angles < > mean they enclose a pseudoknot, and ignore that region from the energy model.
120// I'm assuming the following are true about structure:
121// - Regions with x are always inside parentheses or angle brackets: (xxxx) or <xxxx>.
122// - (..xx) or <...> or <...((..))> is fine too, I ignore everything that's inside <>
123// and I ignore everything that's inside a hairpin loop if it has at least one x.
124// - For a case like: (..(...)[[)]], this function will be called 3 times,
125// so structure can never be: (..(...)xx)
126// - An angle pair is always nested within parentheses: you can have this: (<xxx>) but not this (<)>
127// - If an angle pair is inside a multi-loop, consider it is a multi-loop branch
128// Example: (((...(((...)))..<xxxxxxx>.)))
129// - If an angle pair is nested inside another base pair, consider it is a multi-loop with 2 branches.
130// Example: ..(((...<xxxxxxx>..)))
131// - A multi-loop branch of type <xxxx> contributes 1 branch penalty
132//
133// f is the free value of the energy function c' x + f
134// If reset_counter is 1, then counter[i] is set to 0 for all i, 0 <= i < num_params.
135// Otherwise, the new values get added to whatever was in counter.
136// If ignore_dangles is 1, then dangling ends are not included anywhere
137// Otherwise, dangling ends are included whereever needed only in the parts containing parentheses and dots.
138// For example, in the case (..<xxx>..), dangling ends are included near the parentheses, but not near the angles.
139// If ignore_AU is 1, then AU_penalty is not added at the end of the stem.
140// Otherwise, they are.
141// Returns the value of the energy function
142// Added on Feb 28, 2008
143// Modification on May 26, 2008: if c is NULL, then it doesn't compute the c and f, it only returns the energy
144
